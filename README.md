Project Description 

This project proposes the design and implementation of a Smart Power Monitoring and Protection System using embedded microcontrollers. The system will continuously measure voltage, current, and power consumption of an electrical load and automatically disconnect the load when predefined safety thresholds are exceeded. This functionality mimics an intelligent circuit breaker and is intended to protect components from overcurrent and overpower conditions. 

The system integrates: 

STM32 microcontroller for ADC sampling and control 

Voltage sensing circuit 

Current sensing sensor & circuit 

Optocoupler Relay for load protection 

Led Indicators (Normal, Warning, Fault) 

The goal is to mimic an intelligent circuit breaker capable of real‑time monitoring and hardware‑level protection.

Background 

ACS712 Hall‑Effect Current Sensor Module 

The ACS712 is a Hall‑effect–based current sensing module used to measure both AC and DC current. This project uses the ACS712ELC‑5A variant, which provides a measurement range of ±5 A with a sensitivity of 100 mV/A. When no current is present, the sensor outputs a voltage equal to VCC/2, allowing the microcontroller to detect current direction and magnitude through ADC sampling. 

The sensor operates by measuring the magnetic field generated by current flowing through its internal conductor. Because the measurement is magnetic rather than resistive, the ACS712 provides galvanic isolation, low power loss, and reduced heat generation. These characteristics make it suitable for power monitoring, motor control, and protection systems. 
 
	 

 

 

 

 [insert image]

 

 

 

 

In this project, the ACS712 is placed in series with the load. Its analog output is read by the STM32 ADC, where the firmware converts the voltage into a current estimate. This current measurement is used for real‑time power calculation and overcurrent protection. To ensure accuracy, the sensor is mounted away from external magnetic fields, and software averaging is applied to reduce noise.  

Although the ACS712 provides isolated current measurements, its performance in this project was limited by the characteristics of the load. The system used a small 5 V DC motor, which drew relatively low current during normal operation. Because the ACS712‑5A variant is optimized for higher current ranges, the sensor’s output voltage changed only slightly in response to the motor’s current draw. This made readings more susceptible to electrical noise. Furthermore, the sensor’s inherent offset and sensitivity variations meant that calibration was necessary to establish an accurate zero‑current reference and improve measurement precision. These factors collectively made low‑current measurement more challenging than expected. 

 

Optocoupled 3.3V Relay Module 

A 1‑channel 3.3V relay module with optocoupler isolation is used as the system’s primary protection actuator. The relay supports 10 A at 250 VAC or 10 A at 30 VDC, making it suitable for switching moderate‑power loads. The module includes a Normally Open (NO) contact configuration, a flyback diode for coil protection, and screw‑terminal connectors for secure wiring. 

 

 

 

 [insert image]

 

 

 

The optocoupler provides electrical isolation between the STM32 control signal and the relay coil, improving noise immunity and protecting the microcontroller from voltage transients. When the STM32 outputs a high‑level signal, the relay coil energizes and opens the circuit, disconnecting the load during fault conditions such as overcurrent or overpower events. This relay replaces earlier MOSFET‑based switching concepts with a simpler, safer, and more robust solution for load isolation. 
 
 

STM32F303K8T6 Microcontroller 

The STM32F303K8T6 is a 32‑bit ARM Cortex‑M4 microcontroller used as the central processing unit for sensing, computation, and protection control within the system. It operates at 72 MHz and includes a high‑resolution 12‑bit analog‑to‑digital converter, which allows it to accurately sample the voltage and current signals provided by the measurement circuitry. The microcontroller’s 3.3 V GPIO outputs are fully compatible with the optocoupled relay module, enabling reliable actuation of the protection mechanism during abnormal operating conditions.

Voltage Sensing 

The voltage‑sensing subsystem was implemented using the STM32F303K8T6 microcontroller’s onboard 12‑bit ADC. Because the ADC can only measure voltages between 0–3.3V, we used a voltage divider to scale the load voltage into a safe measurable range. The STM32 then converted this scaled voltage into a digital value, which we processed in firmware to recover the actual load voltage and transmit it to MobaXterm.

Warning System 

The warning system for our Smart Power Monitoring and Protection System was implemented using a three‑LED status indicator connected to the STM32F303K8T6. Each LED provided a real‑time visual representation of the system’s operating state, allowing the user to quickly interpret voltage, current, and power conditions without relying solely on numerical values in MobaXterm. 

The LED color scheme was defined as follows: 

Green LED – Normal Operation   

Illuminates when all measured parameters remain below the warning threshold. This indicates stable and safe system behavior. 

 

Yellow LED – Warning State   

Activates when voltage, current, or power readings approach the predefined safety limits. This provides an early indication that the system is nearing a fault condition. 

 

Red LED – Fault Condition   

Turns on when the STM32 detects an overcurrent or overpower event. In this state, the microcontroller also disables the MOSFET relay to disconnect the load and prevent damage. 

 

Results 

Test Cases 

 

To validate the performance of our Smart Power Monitoring and Protection System, we tested the device using a 5V DC motor connected through a potentiometer. This setup allowed us to gradually increase the voltage supplied to the motor, which in turn increased the current draw and overall power consumption. By slowly adjusting the potentiometer, we were able to observe how the system responded as the measured values approached the warning and fault thresholds. 

 

As the motor load increased, the STM32 continuously updated the voltage, current, and power readings and transmitted them to MobaXterm. This made it possible to monitor the system in real time while watching the LED indicators transition between normal, warning, and fault states. Once the power level exceeded the programmed fault threshold, the system correctly activated the red LED and disconnected the load through the relay. This confirmed that the protection logic operated as intended under controlled and repeatable conditions. 

 

Complications 

Current Sensor Noise 

The shunt‑based current sensor produced noisy readings during testing, especially when the motor was running. The rapid changes in motor current and the electrical noise generated by the brushes caused fluctuations in the ADC values, which made the current measurement appear unstable in MobaXterm. We reduced some of this variation by using software averaging on the STM32, which helped smooth out the rapid spikes and produced a more consistent reading. 

 

Future improvements to stabilize the current measurement could include adding a capacitor in parallel with the load to help reduce high‑frequency noise generated by the motor. If the system continues to measure small loads, converting the readings from amps to milliamps would also improve resolution and accuracy by making better use of the ADC’s available range. 

Insufficient Load for Testing 

One complication we encountered was that our 5V DC motor did not draw enough current to reach the warning or fault thresholds set in our firmware. The motor’s steady‑state current was too low, and even the startup inrush was not high enough to activate the protection logic. This made it difficult to verify that the system could correctly transition into warning and fault states during normal operation. 

This issue could be improved by using a higher power load that naturally draws more current, such as a larger DC motor or a higher wattage resistive load. Another option would be to use an adjustable electronic load so we can precisely control the current during testing. Adjusting the threshold values through software instead of hardcoding them would also make it easier to test different load conditions without modifying the firmware. 
 

 

Future Improvements 

Several enhancements could be made to expand the functionality and reliability of the Smart Power Monitoring and Protection System. One improvement would be the addition of an LCD or OLED display, which would allow the system to show real‑time voltage, current, and power values directly on the device without relying on a computer connection. This would make the system more portable and easier to use in standalone environments. 

Another improvement would be the implementation of wireless monitoring through Bluetooth or WiFi. This would allow the system to transmit data to a phone or laptop without a wired connection, making it more flexible for field testing or remote monitoring applications. Wireless communication would also make it possible to integrate the system into larger IoT‑based power management platforms. 

Current sensing accuracy could also be improved. Adding a capacitor in parallel with the load would help reduce high‑frequency noise generated by motors or switching events, resulting in a more stable current reading. Additionally, switching the measurement units from amps to milliamps would increase resolution when working with small loads, allowing the ADC to capture finer changes in current. 

Finally, adding data logging capabilities would allow the system to record voltage, current, and power values over long periods of time. This would be useful for analyzing trends, identifying intermittent faults, and validating system performance under different load conditions. Logged data could be stored on an SD card or transmitted wirelessly to a database for long‑term analysis. 

 

Conclusion 

The Smart Power Monitoring and Protection System successfully demonstrated the core functionality of an intelligent circuit‑protection device. By integrating voltage sensing, current measurement, real‑time power computation, and an optocoupled relay, the system was able to monitor electrical conditions and disconnect the load when unsafe thresholds were reached. The STM32F303K8T6 provided reliable ADC sampling and fast processing, while the LED indicators offered clear visual feedback for normal, warning, and fault states. Testing with a variable motor load confirmed that the protection logic operated correctly and that the system responded predictably as power levels increased. 

Although the system performed as intended, the project also revealed several areas where performance could be improved. Low current levels from the 5V motor made it difficult to fully exercise the protection thresholds, and noise in the current sensor readings introduced some instability in the measurements. These limitations helped highlight opportunities for refinement, such as improved filtering, higher‑resolution current measurement, and more flexible load testing. 

Overall, the project provided valuable hands‑on experience with embedded systems, analog measurement, and hardware‑based protection. The system forms a strong foundation for future enhancements, including display integration, wireless monitoring, improved sensing accuracy, and long‑term data logging. With these additions, the design could evolve into a more capable and versatile power‑monitoring platform suitable for a wide range of applications.
